## 5 ÉTAPES SIMPLES

### ÉTAPE 1: Installer 3 trucs
```
Allez sur:
  → https://git-scm.com (télécharger)
  → https://code.visualstudio.com (télécharger)
  → https://nodejs.org (télécharger)

Cliquez "Download", installez, acceptez tout par défaut
✅ C'est fait
```

**Temps: 15 minutes**

---

### ÉTAPE 2: Créer un compte GitHub
```
Allez sur: https://github.com
Cliquez "Sign up"
Remplissez vos infos
Vérifiez votre email
✅ Vous avez un compte GitHub
```

**Temps: 5 minutes**

---

### ÉTAPE 3: Créer votre dossier et télécharger les fichiers
```
1. Créez un dossier sur votre Bureau:
   Nommez-le: suivi-maternelle

2. Téléchargez tous les fichiers (42 fichiers)

3. Mettez-les dans le bon dossier:
   Regardez: ARBORESCENCE_EXACTE.md
   C'est comme une carte!

✅ Vous avez la structure
```

**Temps: 1 heure**

---

### ÉTAPE 4: Mettre sur GitHub
```
1. Ouvrez VS Code
2. Cliquez File → Open Folder
3. Sélectionnez votre dossier suivi-maternelle
4. Ouvrez Terminal (Terminal → New Terminal)
5. Tapez ces 4 lignes une par une:

   git init
   git add .
   git commit -m "Mon projet"
   git remote add origin https://github.com/VOTRE_USERNAME/suivi-maternelle.git
   git branch -M main
   git push -u origin main

✅ C'est sur GitHub!
```

**Important:** Remplacez `VOTRE_USERNAME` par votre vrai username GitHub!

**Temps: 10 minutes**

---

### ÉTAPE 5: Tester
```
1. Allez sur GitHub.com
2. Allez sur votre repo suivi-maternelle
3. Vous voyez tous vos fichiers? ✅

Maintenant, vous pouvez:
  - Modifier votre code
  - Faire: git add . → git commit → git push
  - Ça se sauvegarde sur GitHub tout seul!
```

**Temps: 5 minutes**

## 📝 COMMANDES À RETENIR

Si vous oubliez tout, ces 4 commandes suffisent:

```

git commit -m "message"        (Sauvegarder)
git push                       (Envoyer à GitHub)
git pull                       (Télécharger depuis GitHub)
```

Vous pouvez copier-coller! 😊

## 🔑 POINTS CLÉS

### ✅ À FAIRE
- Créer la bonne structure (voir `ARBORESCENCE_EXACTE.md`)
- Mettre les fichiers au bon endroit
- Remplacer `VOTRE_USERNAME` par votre vrai username

### ❌ À NE PAS FAIRE
- Envoyer le dossier `node_modules/` sur GitHub (le `.gitignore` le fait automatiquement)
- Oublier de remplacer votre username
- Placer les fichiers au mauvais endroit
